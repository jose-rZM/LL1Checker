{\footnotesize\ttfamily void lexer\+::tokenize (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [private]}}



Tokenizes the input file using Boost Spirit Lex. 

This function reads the content of the file specified by {\ttfamily filename\+\_\+}, tokenizes it using Boost Spirit Lex, and stores the resulting tokens in the {\ttfamily tokens\+\_\+} member variable. If the tokenization process encounters an invalid token, a {\ttfamily \doxylink{classLexerError}{Lexer\+Error}} is thrown with an error message indicating the invalid token.


\begin{DoxyExceptions}{Exceptions}
{\em \doxylink{classLexerError}{Lexer\+Error}} & If an invalid token is encountered during tokenization.\\
\hline
\end{DoxyExceptions}
The function performs the following steps\+:
\begin{DoxyEnumerate}
\item Opens the file specified by {\ttfamily filename\+\_\+} and reads its content into a string.
\item Converts the string into a C-\/style string (char array) for processing.
\item Uses Boost Spirit Lex to tokenize the input string.
\item If tokenization is successful, the tokens are stored in the {\ttfamily tokens\+\_\+} member variable.
\item If tokenization fails (e.\+g., due to an invalid token), a {\ttfamily \doxylink{classLexerError}{Lexer\+Error}} is thrown.
\end{DoxyEnumerate}

\begin{DoxyNote}{Note}
The function relies on the {\ttfamily \doxylink{structlexer_1_1parse__input}{parse\+\_\+input}} functor and the {\ttfamily add} function to handle the tokenization and token storage, respectively.
\end{DoxyNote}
