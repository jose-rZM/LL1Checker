Computes the FOLLOW sets for all non-\/terminal symbols in the grammar. 

The FOLLOW set of a non-\/terminal symbol A contains all terminal symbols that can appear immediately after A in any sentential form derived from the grammar\textquotesingle{}s start symbol. Additionally, if A can be the last symbol in a derivation, the end-\/of-\/input marker ({\ttfamily \$}) is included in its FOLLOW set.

This function computes the FOLLOW sets using the following rules\+:
\begin{DoxyEnumerate}
\item Initialize FOLLOW(\+S) = \{ \$ \}, where S is the start symbol.
\item For each production rule of the form A → α\+Bβ\+:
\begin{DoxyItemize}
\item Add FIRST(β) (excluding ε) to FOLLOW(\+B).
\item If ε ∈ FIRST(β), add FOLLOW(\+A) to FOLLOW(\+B).
\end{DoxyItemize}
\item Repeat step 2 until no changes occur in any FOLLOW set.
\end{DoxyEnumerate}

The computed FOLLOW sets are cached in the {\ttfamily follow\+\_\+sets\+\_\+} member variable for later use by the parser.

\begin{DoxyNote}{Note}
This function assumes that the FIRST sets for all symbols have already been computed and are available in the {\ttfamily first\+\_\+sets\+\_\+} member variable.
\end{DoxyNote}

Calculates the FIRST set for a given production rule in a grammar. 

The FIRST set of a production rule contains all terminal symbols that can appear at the beginning of any string derived from that rule. If the rule can derive the empty string (epsilon), epsilon is included in the FIRST set.

This function computes the FIRST set by examining each symbol in the production rule\+:
\begin{DoxyItemize}
\item If a terminal symbol is encountered, it is added directly to the FIRST set, as it is the starting symbol of some derivation.
\item If a non-\/terminal symbol is encountered, its FIRST set is recursively computed and added to the result, excluding epsilon unless it is followed by another symbol that could also lead to epsilon.
\item If the entire rule could derive epsilon (i.\+e., each symbol in the rule can derive epsilon), then epsilon is added to the FIRST set.
\end{DoxyItemize}

Parses an input string or file using the LL(1) parsing algorithm. 

This function performs syntactic analysis on the input based on the LL(1) parsing table, working to validate whether the input string conforms to the grammar. The parsing process involves a stack-\/based approach, where\+:


\begin{DoxyItemize}
\item The function initializes a stack with the starting symbol of the grammar.
\item For each symbol in the input, it matches and expands according to the entries in the LL(1) parsing table.
\item If a match is found for the current input symbol and top of the stack, the function advances in the input and continues parsing.
\item If a production rule applies, it expands the non-\/terminal on the stack using the rule.
\item If an unexpected symbol or parsing conflict arises, the function returns {\ttfamily false}, indicating that the input does not conform to the grammar.
\end{DoxyItemize}

The function returns {\ttfamily true} if parsing completes successfully, reaching the end of the input and stack without errors. Otherwise, it returns {\ttfamily false}.

\begin{DoxyReturn}{Returns}
{\ttfamily true} if the input is parsed successfully, meaning it conforms to the LL(1) grammar; {\ttfamily false} if parsing fails due to a mismatch, conflict, or unexpected input symbol. 
\end{DoxyReturn}
